# 增强交互与业务功能 Spec

## Why
当前"星门塔罗"已具备完整的3D手势抽牌、正逆位、三张牌阵、音效和魔法阵功能，但缺少两个关键维度：**结果的传播性**（用户无法分享自己的塔罗结果）和**日常回访动机**（用户抽完一次后没有理由再次打开）。此外，魔法阵功能存在代码缺陷（`appState.handY` 未定义）导致永远无法触发，摄像头在重置时未释放资源。

## What Changes
- 修复魔法阵检测中 `handY` 未定义的 bug，使魔法阵特效可正常触发
- 修复手动模式下摄像头预览窗仍显示空白区域的问题
- 新增"分享塔罗结果"功能：结果面板展示时，可将当前牌面信息渲染为一张精美的分享图片，支持保存到本地
- 新增"每日一牌"功能：每天首次打开时自动抽取当日专属牌，作为独立于正式占卜的轻量互动入口

## Impact
- Affected specs: 新增 2 个用户功能 + 2 个 bug 修复
- Affected code: `index.html`（CSS、HTML 结构、JavaScript 逻辑）

## ADDED Requirements

### Requirement: 分享塔罗结果图片
系统 SHALL 在单张牌结果展示后，提供"生成分享图"按钮，点击后将当前牌面信息渲染为一张固定尺寸（1080×1920）的 Canvas 图片，包含：牌名、正逆位、牌义、用户问题（如有）、应用品牌标识。渲染完成后弹出全屏预览，用户可长按/右键保存。

#### Scenario: 生成分享图成功
- **WHEN** 用户在单张牌结果面板中点击"分享"按钮
- **THEN** 系统在后台绘制一张 1080×1920 的 Canvas 图片，包含深色星空背景、牌名大字、正逆位标识、牌义文字、底部品牌文字"星门塔罗"
- **AND** 弹出全屏预览层显示该图片
- **AND** 预览层底部显示"保存图片"按钮和"关闭"按钮

#### Scenario: 三张牌阵不支持分享
- **WHEN** 当前为三张牌阵结果
- **THEN** 不显示"分享"按钮（牌阵信息复杂度超出单图表达范围）

#### Scenario: 保存图片
- **WHEN** 用户在预览层点击"保存图片"
- **THEN** 系统通过 Canvas.toDataURL("image/png") 生成图片并触发浏览器下载（文件名：星门塔罗-{牌名}-{日期}.png）

### Requirement: 每日一牌
系统 SHALL 在用户每天首次打开应用时，在开场弹窗中展示"今日塔罗"卡片，自动抽取一张当日专属牌（基于日期的确定性随机，同一日期始终得到相同结果）。用户可选择查看解读后直接进入正式占卜，也可跳过。

#### Scenario: 首次打开当天
- **WHEN** 用户在当天首次打开应用
- **THEN** 开场弹窗中在问题输入区上方显示一张"今日塔罗"卡片
- **AND** 卡片显示：当日牌名、正逆位、一段简短的牌义摘要
- **AND** 卡片底部有"查看详情"按钮（展开完整解读）和"开始占卜"按钮

#### Scenario: 同一天再次打开
- **WHEN** 用户在当天非首次打开应用
- **THEN** 仍然显示"今日塔罗"卡片，且与首次打开时抽到的牌一致（基于 localStorage 缓存）

#### Scenario: 跳过每日一牌
- **WHEN** 用户点击"开始占卜"
- **THEN** "今日塔罗"卡片收起/隐藏，进入正常的占卜流程

#### Scenario: 查看详情
- **WHEN** 用户点击"查看详情"
- **THEN** 卡片展开显示完整的关键词和行动建议
- **AND** 按钮变为"开始占卜"

### Requirement: 每日一牌确定性算法
系统 SHALL 使用 `YYYY-MM-DD` 字符串作为种子，通过简易哈希算法生成确定性索引，从22张大阿卡那中选取当日牌。逆位判定同样基于日期哈希，非随机。

#### Scenario: 同日期一致性
- **WHEN** 两次打开使用相同的 `YYYY-MM-DD`
- **THEN** 得到完全相同的牌和正逆位结果

## MODIFIED Requirements

### Requirement: 开场弹窗布局
开场弹窗 SHALL 在"今日塔罗"卡片显示时，将其置于问题输入区上方，整体布局保持居中、自上而下排列。

## REMOVED Requirements
无

## Bug Fixes

### Bug 1: 魔法阵 handY 未定义
`appState` 对象中未定义 `handY` 字段，导致 `updateMagicCircle()` 中 `appState.handY * 0.5 + 0.25` 计算结果为 `NaN`，魔法阵检测永远无法触发。
- **修复方案**：在 MediaPipe 手部追踪回调 `onHandResults` 中，同步更新 `appState.handY = landmarks[0].y`（手腕 landmark 的 Y 坐标）

### Bug 2: 手动模式摄像头预览窗
用户选择"直接抽牌"时，摄像头预览窗 `.video-wrap` 仍以空黑区域显示。
- **修复方案**：在手动模式启动时（`startManualMode` 或 `manualBtn` 点击事件），为 `.video-wrap` 添加 `display: none` 或通过 CSS class 控制隐藏
