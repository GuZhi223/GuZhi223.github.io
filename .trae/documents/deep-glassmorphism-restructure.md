# 深度重构计划：iOS 毛玻璃界面全面重构

## 背景

上一轮我们将 CSS 换成了毛玻璃风格（深色背景 + 毛玻璃卡片 + 流光球），但 HTML 结构基本保持不变——只是在原有元素上加了 `.glass` class。用户希望的是**更彻底的界面重构**，不仅仅是换皮，而是从布局、信息层级、交互节奏上全面改造。

## 当前问题分析

| 问题 | 说明 |
|---|---|
| **布局单调** | 所有 section 都是"标题行 + 网格卡片"的重复模式，视觉节奏单一 |
| **Hero 不够沉浸** | 左右分栏的介绍+状态卡片，和原版白底时没有本质区别 |
| **"关于站点"三卡片无存在感** | 三张静态卡片缺乏视觉重量，可以考虑用更有趣的形式呈现 |
| **作品卡片同质化** | 5+3 张 work 卡片全部用相同模板，没有区分度 |
| **section 之间缺乏视觉分隔** | 在深色背景上各 section 粘连在一起 |
| **滚动动画过于简单** | 只有统一的淡入上滑，没有交错和节奏感 |
| **导航是胶囊但没有活跃状态** | 滚动时无法知道当前在哪个 section |

## 重构方案

### 1. Hero 区域：全屏沉浸式首屏

**改动**：去掉左右分栏，改为全宽居中的大字排版，营造"落地页首屏"的沉浸感。

- eyebrow + h1 + lead 居中排列，占满视口高度（`min-height: 100vh`）
- 三个状态信息从右侧卡片改为 h1 下方的横向 pills 条（类似 iOS 的 status chips）
- 两个 CTA 按钮居中放置在状态条下方
- 光球在 Hero 区域最为明显，往下逐渐减弱

### 2. "关于站点"：删除独立 section，融入 Hero

**改动**：将"关于站点"的三卡片信息整合到 Hero 区域底部作为三个 feature highlight pills，不再单独占一个 section。这样减少了一个重复的"标题+网格"块。

### 3. 作品展示：非均匀网格 + 视觉层次

**改动**：将"一些分享"和"开源库"的作品卡片改为不均匀网格布局，给不同作品以不同视觉权重。

- 使用 CSS Grid 的 `grid-template-areas` 或 span 实现第一张卡片大、其余小的"杂志式"布局
- 卡片内部：icon 移到左上角作为彩色渐变 badge
- 标签移到卡片底部
- hover 效果增强：卡片内部出现微妙的渐变光晕跟随

### 4. Section 之间：添加视觉分隔线和呼吸空间

**改动**：
- 每个 section 之间加入一个细长的发光分割线（渐变透明的水平线）
- section padding 加大，给内容更多呼吸空间
- 每个 section 的 h2 左侧加一条竖向彩色 accent bar

### 5. 联系区域：居中单面板

**改动**：将两栏联系信息合并为一个居中的大玻璃面板，内部用行式布局展示 GitHub 和邮箱，更紧凑。

### 6. 导航：滚动高亮活跃状态

**改动**：JS 添加 Intersection Observer 监听各 section，当 section 进入视口时，导航栏中对应的链接高亮（白色文字 + 背景高亮）。

### 7. 动画增强：交错入场 + 视差

**改动**：
- 卡片组使用交错动画（stagger），每张卡片延迟 0.1s 依次入场
- hero 的光球添加视差效果：滚动时以不同速度移动
- 页面底部的光球颜色更暖（橙/粉），顶部偏冷（蓝/紫），形成渐变氛围

### 8. Footer：玻璃底栏

**改动**：footer 改为固定在底部的窄条毛玻璃栏，与顶部导航呼应。

## 具体实施步骤

### Step 1: 重写 HTML 结构
- 重构 Hero 为全宽居中沉浸式
- 将"关于站点"三卡片融入 Hero 底部
- 重新组织作品卡片 HTML（添加 data 属性控制 span）
- 合并联系区域为单面板
- 为 section 添加 accent line 和 divider 元素
- 为导航添加 `data-section` 属性
- 为 footer 改为 glass 样式

### Step 2: 重写 CSS
- Hero 全屏居中排版样式
- Status pills 横向排列
- Feature highlights 横向 pill 条
- 非均匀网格 `grid-template-areas`
- Section 分隔线和 accent bar 样式
- 交错动画的 `transition-delay`
- 导航活跃状态样式
- Footer 固定底栏
- 视差滚动相关样式

### Step 3: 重写 JS
- 导航活跃状态 Intersection Observer
- 光球视差滚动效果
- 交错 reveal 动画（为每个卡片设置递增 delay）
- 保留邮箱复制逻辑

### Step 4: 验证
- 所有文本内容完整
- 所有链接完好
- 邮箱复制正常
- 响应式适配（860px / 560px）
- 滚动动画流畅
- 导航高亮正确

## 需要保留的不变项

- 所有文字内容（标题、描述、标签文字、链接地址、邮箱）
- 邮箱复制 JS 逻辑
- Font Awesome CDN 引用
- 响应式断点（860px / 560px）
